# Secure Delete Overview

> 현재 문서는 Phase 0 기준이며 실제 구현이 아니라 설계/범위/거버넌스 문서입니다. 이 문서는 OS 기능, 백신, Secure Delete, AI WebUI, 커널, 드라이버, ISO 빌드를 구현하지 않습니다.

## 정의

Secure Delete는 anti-forensics가 아니라 사용자 소유 데이터 보호 기능입니다. 목적은 사용자가 본인 소유 데이터의 노출 위험을 줄이도록 돕는 것입니다.

## Phase 0 범위

Phase 0에서는 실제 삭제 기능을 구현하지 않습니다. 다음 항목만 설계합니다.

- 삭제 대상 preview
- dry-run 결과 표시
- 명시적 사용자 확인
- 저장장치와 파일시스템별 한계 고지
- rollback 불가능성 고지
- 감사 로그 정책
- 암호화 기반 삭제와 key destruction 후보 검토

## 안전 문구 기준

Secure Delete는 100% 물리적 복구 불가능성을 주장하지 않습니다.

권장 표현:

> 현재 저장장치, 파일시스템, 암호화 상태에서 가능한 가장 강한 안전 삭제 절차를 완료했습니다.

## 금지 범위

- anti-forensics 목적 기능
- 불법 증거 훼손 목적 기능
- 검증 없는 완전 삭제 기능
- 사용자 확인 없는 삭제
- 대상 범위 preview 없는 삭제
- Phase 0에서 실제 삭제 실행
