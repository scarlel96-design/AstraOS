# AstraOS Coding Standards

> 현재 문서는 Phase 0 기준이며 실제 구현이 아니라 설계/범위/거버넌스 문서입니다. 이 문서는 OS 기능, 백신, Secure Delete, AI WebUI, 커널, 드라이버, ISO 빌드를 구현하지 않습니다.

## 적용 범위

이 문서는 이후 Phase에서 AstraOS 코드와 문서를 작성할 때 적용할 기본 기준입니다. Phase 0에서는 실제 기능 구현이 아니라 기준과 원칙만 정의합니다.

## 언어 및 설명 기준

- 소스 코드 내부 주석과 기술 설명은 한국어로 작성합니다.
- 변수명, 함수명, 타입명, 파일명은 글로벌 표준 명명법을 따릅니다.
- 사용자 인터페이스와 사용자 메시지는 코드에 하드코딩하지 않습니다.
- UI 문구는 한국어와 영어를 모두 지원할 수 있는 리소스 구조를 전제로 설계합니다.

## 다국어 구조

이후 구현에서는 KO/EN 다국어 구조를 기본값으로 둡니다.

권장 원칙:

- 사용자 표시 문자열은 리소스 파일, JSON dictionary, locale table 등으로 분리합니다.
- 로그, 오류, 확인 문구는 사용자용 메시지와 개발자용 진단 메시지를 구분합니다.
- destructive 작업 확인 문구는 한국어와 영어 모두에서 의미가 동일해야 합니다.
- Secure Delete, Astra Shield, AI Studio 같은 보안 기능의 용어는 문서 전체에서 일관되게 유지합니다.

## 오류 처리와 널 방어

이후 구현에서는 예외 처리와 널 참조 방어를 기본으로 포함합니다.

기본 원칙:

- 외부 입력은 신뢰하지 않습니다.
- 파일 경로, 네트워크 응답, 설정 값, 권한 값, 사용자 선택 값은 null 또는 비어 있을 수 있다고 가정합니다.
- 실패는 사용자에게 안전하고 이해 가능한 방식으로 보고합니다.
- 실패 시 destructive 작업은 중단하고 안전 상태를 유지합니다.
- 복구 가능한 오류와 복구 불가능한 오류를 구분합니다.

## 위험 작업 구현 기준

위험 작업은 다음 작업을 포함합니다.

- 파일 삭제
- 권한 변경
- 네트워크 공개
- 부팅 구성 변경
- 시스템 서비스 변경
- 암호화 또는 복호화
- 보안 정책 변경
- 백업, 복구, rollback 조작

위험 작업은 다음 순서가 기본입니다.

1. dry-run
2. preview
3. 사용자 확인
4. 실행 전 로그
5. 실행
6. 결과 검증
7. rollback 또는 복구 안내

Phase 0에서는 이 순서를 기준으로만 문서화하며 실제 실행 코드는 작성하지 않습니다.

## 보안 기능 코딩 경계

### Secure Delete

Secure Delete는 anti-forensics가 아니라 사용자 소유 데이터 보호 기능입니다. 구현이 필요한 이후 Phase에서도 검증 없는 완전 삭제 주장이나 불법 증거 훼손 목적의 기능은 허용하지 않습니다.

### Astra Shield

Astra Shield는 공격 기능이 아니라 백신/EDR/복구 구조입니다. 공격 코드, exploit code, credential theft, 백신 우회, 악성코드, evasion 로직은 작성하지 않습니다.

### Astra Shield Guardian

Guardian은 Astra Shield 자체 보호와 자동 복구 구조입니다. 은닉 persistence가 아니라 visible하고 reversible한 복구 흐름을 기준으로 합니다.

### AI Studio

AI Studio는 WebUI 외부 공개가 아니라 sandbox/localhost 기본값입니다. 외부 공개, API key 접근, 파일 접근, 네트워크 접근은 명시적 권한과 사용자 확인을 요구하는 방향으로 설계합니다.

## Phase 0 문서 작성 기준

- 구현 가능성을 과장하지 않습니다.
- 실제 기능이 존재하는 것처럼 표현하지 않습니다.
- 금지 범위는 명확히 적습니다.
- TODO는 다음 Phase 0 작업 또는 이후 Phase 작업으로 분리합니다.
- 보안-sensitive 결정에는 목적, 범위, 오용 가능성, 완화책을 함께 적습니다.

## 검증 기준

문서 작업의 기본 검증은 다음과 같습니다.

- 생성 파일 목록 확인
- 핵심 용어 검색
- Phase 0 범위 확인
- 금지 구현 내용 포함 여부 확인
- `git status --short` 확인
